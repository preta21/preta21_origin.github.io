Verilog实验分析1-按键消抖模块设计

2021/9/4

实验背景

按键被用户按下或者松开时会产生抖动，电平不稳定。

实验环境

Alinx 7325 FPGA 开发板、Vivado 2020.2、idea/vscode(美观的代码编辑)

实验目的和内容

设计 Verilog 程序，通过计数计时的方式来判断按键被按下或者松开时是否稳定，是否真的为低电平或者高电平。
按键按下时，判断为电平稳定后，程序中计数器加一，通过开发板上的led灯来显示计数结果。

程序设计

程序设计中分为计数模块、消抖模块和综合模块三部分。
计数模块是用来对按键出发的稳定电平变化进行计数的。每次成功的按键变化会触发计数模块加一，从0开始，
该结果实时更新在开发板的四个led灯上。也就是输出实验现象。
消抖模块是用来判断按键是否稳定的；
综合模块最后统一对消抖模块和计数模块进行了综合调用。实现实验功能。

计数模块的逻辑最为简单，代码如下：


/**
* @Des: 按键稳定时间（保持时间）计数（10ms here）
        代码中变量 data 的自增 1 应当是 1ms
* @Author: preta
* @Date: 2021-9-5
*/

module count_m10 (
    input clk,
    input rst_n,
    input en, // 使能信号
    input clr, // 计数同步复位
    output reg[3:0] data, // 计数值
    output reg t // 寄存使能信号
);

    always @ (posedge clk or negedge rst_n)
        begin
            if (rst_n == 0)
                begin
                    data <= 4'd0;
                    t <= 1'd0;
                end
            else if (clr)
                begin
                    data <= 4'd0;
                    t <= 1'd0;
                end
            else if (en)
                begin
                    if (data == 4'd9)
                        begin
                            t <= 1'b1; // 计数到9（0~9）——对应输出到 key_debounce.v模块中，led 显示计数结果
                            data <= 4'd0;
                        end
                    else
                        begin
                            t <= 1'b0;
                            data <= data + 4'd1;
                        end
                end
            else
                t <= 1'b0;
        end

endmodule

然后是消抖模块代码

`timescale 1ns/100ps

/**
* @Des: 按键消抖模块
* @Author: preta
* @Date: 2021-9-4
*/

module ax_debounce (
    input clk,
    input rst,
    input button_in,
    output reg button_posedge,
    output reg button_negedge,
    output reg button_out
);

    parameter N = 32; // 计时器位宽
    parameter FREQ = 200; // 时钟频率单位：MHz 频率(frequency)
    parameter MAX_TIME = 20; // 单位：ms，抖动判断时间，20ms？
    localparam TIMER_MAX_VAL = MAX_TIME * 1000 * FREQ; // ms(10^-3) * MHz(10^6) = 10^3 = 1000; 这里的值应该是最防抖过程中的最大计数次数

    reg [N-1:0] q_reg; //
    reg [N-1:0] q_next;
    reg DFF1, DFF2; // 输入 D 触发器
    wire q_add; // 标志位
    wire q_reset;
    reg button_out_d0;

    assign q_reset = (DFF1 ^ DFF2); // 异或运算，同则为0，不同则为1
    assign q_add = ~(q_reg == TIMER_MAX_VAL); // reg计数到 TIMER_MAX_VAL 则停止计数；


    always @ (q_reset, q_add, q_reg)
        begin
            case ({q_reset, q_add})
                2'b00:
                    q_next <= q_reg;
                2'b01:
                    q_next <= q_reg + 1;
                default :
                    // 复位
                    q_next <= {N{1'b0}};
            endcase
        end

    // 触发器输入，q值更新
    always @ (posedge clk or posedge rst)
        begin
            if (rst == 1'b1)
                begin
                    DFF1 <= 1'b0;
                    DFF2 <= 1'b0;
                    q_reg <= {N{1'b0}};
                end
            else
                begin
                    DFF1 <= button_in;
                    DFF2 <= DFF1;
                    q_reg <= q_next;
                end
        end

    // 计数控制 button_out
    always @ (posedge clk or posedge rst)
        begin
            if (rst == 1'b1)
                button_out <= 1'b1;
            else if (q_reg == TIMER_MAX_VAL)
                button_out <= DFF2;
            else
                button_out <= button_out;
        end

    // 计数控制 button_out_d0
    always @ (posedge clk or posedge rst)
        begin
            if (rst == 1'b1)
                begin
                    button_out_d0 <= 1'b1;
                    button_posedge <= 1'b0;
                    button_negedge <= 1'b0;
                end
            else
                begin
                    button_out_d0 <= button_out;
                    button_posedge <= ~button_out_d0 & button_out;
                    button_negedge <= button_out_d0 & ~button_out;
                end
        end

endmodule

module key_debounce (
    input sys_clk_p,
    input sys_clk_n,
    input rst_n,
    input key,
    output [3:0] led
);

    wire sys_clk;
    IBUFGDS u_ibufg_sys_clk (
        .I (sys_clk_p),
        .IB (sys_clk_n),
        .O (sys_clk)
    );

    wire button_negedge; // 键下降沿
    ax_debounce ax_debounce_m0 (
        .clk (sys_clk),
        .rst (~rst_n),
        .button_in (key),
        .button_posedge (),
        // 只获取按键的下降沿信号（按键按下），这里只对这个接口进行了赋值
        .button_negedge (button_negedge),
        .button_out ()
    );

    wire [3:0] count;
    wire t0;
    count_m10 count10_m0 (
        .clk (sys_clk),
        .rst_n (rst_n),
        .en (button_negedge),
        .clr (1'b0),
        .data (count),
        .t (t0)
    );
    assign led = count;

endmodule

Verilog 代码的思维

2021/9/4

17:05
Verilog 和高级编程语言的不同，我想我现在可以总结上一两点。 

    高级语言的好处显而易见，其思维是无限接近于人的思想的，按照人的思维方式来完成任务。基本上其实所见即所得。
所以其做的事情也更高级，从更高的层面，主要是算法这一思想层面去工作研究。如果为高级语言完成的任务进行一个结构分析，  
那么结果也就是人所想的过程，只不过做一些思维和人性化的美化，是软件工程中进行软件的生命周期分析（需求分析、可行性分析、概要设计、
详细设计、编码、测试、调试），或是在敏捷开发中，对于每一个用户场景（User Story）进行分析从而得出合理的功能流程。
整体下来就是要完成人要做的事情。基本的思路是很明确的。

    最近除了在学习《数字电路》的知识，也在看一些Verilog的经典教程，敲敲代码，读读心得之间，也对Verilog逐渐入门——
当然随着学习的深入，学习的难度也在上升，感觉是自己在给自己上课。也许相对本科时期那种相对宽松的学习时间和环境，
我当今的效率已经很高了，而且投入与产出目前也比较好。但是，我现在是在硕士阶段，等待我做的不光有前所未有的新课程
的学习，还有实际要完成的项目，我应当以一个更高的水准来要求自己。

虽然不是很喜欢所看的Verilog教程的作者的语言风格，但是其中有个概念我觉得很对：Verilog语言的作用不是编码，
而是 建模 。

    就尤其我目前的学习体验来说，最有收获感的学习过程，莫过于刚学习过数字电路的基础知识，学习了一两个逻辑器件，然后使用
Verilog实现器件的逻辑；单纯地去复刻别人的代码，即使有代码注释，也很难读懂代码。所以，特别是现在学习Verilog的初期，我觉得
遵守组合电路或者时序电路的构建思路再来编写代码，很重要。而且，我更深地感觉到，写Verilog的代码，绝不能脱离数字电路这个实体。
Verilog代码只是属于整个数字电路研发过程中的一部分。

    比如，从组合电路出发的话，就是
    1.  描述功能；
    2.  做真值表；
    3.  根据真值表写出布尔表达式；
    4.  化简布尔表达式（卡诺图）；
    5.  根据布尔表达式画出逻辑电路图；
    6.  使用 verilog 对电路进行编码
        ·电路综合（此处可以通过vivado等EDA得到代码对应的逻辑电路图RTL）；
        ·电路布线（得到根据硬件的实际接口情况完成的电路图）；
        ·生成bit流文件；
        ·进行仿真/生成bin文件，bin文件可以烧写到开发板上
    
    从时序电路出发，大概是（有些知识还在学习之中）
    1.  描述功能；
    2.  做状态转换图（Moore / Mealy）；
    3.  做状态转换表；
    4.  对状态进行 二进制编码 或者 独热编码（被编码的一个状态可能对应多位编码，在新的状态转换表中是多位输入）；
    5.  根据编码后的状态更新状态转换表，并做出状态和输出的对应表；
    6.  根据编码后的状态转换表、输出表（此时类似于真值表形式），做出每一个下一个状态和输出的布尔表达式；
    7.  化简布尔表达式（卡诺图）；
    8.  根据布尔表达式画出逻辑电路图；
    9.  使用 verilog 对电路进行编码
        ·电路综合（此处可以通过vivado等EDA得到代码对应的逻辑电路图RTL）；
        ·电路布线（得到根据硬件的实际接口情况完成的电路图）；
        ·生成bit流文件；
        ·进行仿真/生成bin文件，bin文件可以烧写到开发板上

也就是说，Verilog 对应的是电路图，而不是人的思维方式——是和硬件打交道的方式。高级语言尤其是 Java 和 Python 这样的高度友好的语言，思维方式已经几乎和人的思维差不多，在编码时考虑的是业务的逻辑，人的想法；而要想用好 Verilog，就必须要了解电路，理解电路的思维与工作方式，才能用好这门与之交互的语言——人和机器的几乎直接交流。
是的，所有编程语言都应该是一门伟大的艺术——人和机器交互的艺术，交流的艺术，这样的代码才是像基因一样具有灵性的，具有灵魂。
而 Verilog 是如此的直接，即使不显得那么灵敏和柔软，但是却是直击灵魂的。

是的，在向计算机的底层出发，接触人类智慧最深层的灵魂。

Verilog 简单的测试代码注意事项

2021/9/4

在主模块中注意：

原模块中的 output 类型接口可以声明为 reg 类型，但是 input 接口不能声明为 reg;



在测试模块中注意：

1. 在测试模块中，声明的变量一定都是 wire 类型的;

2. 测试模块的模块声明中没有变量;

3. 原模块与测试模块中各个变量所声明的长度必须一致。


示例的代码tmp.v如下：

其中tmp是主模块，tmp_test是测试模块。

module tmp (
    input [1:0] signal,
    output reg [1:0] flag
);
    initial begin // 如果这里使用 always 语句块，则最终输出的总是 x 
        if (signal == 2'd0) begin
            flag = 2'd2;
        end
        else if (signal == 2'd1) begin
            flag = 2'd1;
        end
        else if (signal == 2'd2) begin
            flag = 2'd0;
        end
    end
endmodule //tmp


module tmp_test;
wire [1:0] sig;
wire [1:0] f;
assign sig = 2'd1;
tmp uut (
    .signal(sig),
    .flag(f)
);
initial begin
    $display(f); // 输出结果
end
endmodule // tmp_test

Verilog 中的位运算符和逻辑运算符

位运算符就是与或非运算，a & b, a | b, ~a，就是与或非；

逻辑运算符就是汉语表达中且、或这样的条件组合逻辑。比如(a>b) && (a!=0)这种，
就是(a>b)并且(a!=0)两个条件都满足。
