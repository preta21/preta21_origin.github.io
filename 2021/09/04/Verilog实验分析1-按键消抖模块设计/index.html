Verilog实验分析1-按键消抖模块设计

2021/9/4 verilog

实验背景

按键被用户按下或者松开时会产生抖动，电平不稳定。

实验环境

Alinx 7325 FPGA 开发板、Vivado 2020.2、idea/vscode(美观的代码编辑)

实验目的和内容

设计 Verilog 程序，通过计数计时的方式来判断按键被按下或者松开时是否稳定，是否真的为低电平或者高电平。
按键按下时，判断为电平稳定后，程序中计数器加一，通过开发板上的led灯来显示计数结果。

程序设计

程序设计中分为计数模块、消抖模块和综合模块三部分。
计数模块是用来对按键出发的稳定电平变化进行计数的。每次成功的按键变化会触发计数模块加一，从0开始，
该结果实时更新在开发板的四个led灯上。也就是输出实验现象。
消抖模块是用来判断按键是否稳定的；
综合模块最后统一对消抖模块和计数模块进行了综合调用。实现实验功能。

计数模块的逻辑最为简单，代码如下：


/**
* @Des: 按键稳定时间（保持时间）计数（10ms here）
        代码中变量 data 的自增 1 应当是 1ms
* @Author: preta
* @Date: 2021-9-5
*/

module count_m10 (
    input clk,
    input rst_n,
    input en, // 使能信号
    input clr, // 计数同步复位
    output reg[3:0] data, // 计数值
    output reg t // 寄存使能信号
);

    always @ (posedge clk or negedge rst_n)
        begin
            if (rst_n == 0)
                begin
                    data <= 4'd0;
                    t <= 1'd0;
                end
            else if (clr)
                begin
                    data <= 4'd0;
                    t <= 1'd0;
                end
            else if (en)
                begin
                    if (data == 4'd9)
                        begin
                            t <= 1'b1; // 计数到9（0~9）——对应输出到 key_debounce.v模块中，led 显示计数结果
                            data <= 4'd0;
                        end
                    else
                        begin
                            t <= 1'b0;
                            data <= data + 4'd1;
                        end
                end
            else
                t <= 1'b0;
        end

endmodule

然后是消抖模块代码

`timescale 1ns/100ps

/**
* @Des: 按键消抖模块
* @Author: preta
* @Date: 2021-9-4
*/

module ax_debounce (
    input clk,
    input rst,
    input button_in,
    output reg button_posedge,
    output reg button_negedge,
    output reg button_out
);

    parameter N = 32; // 计时器位宽
    parameter FREQ = 200; // 时钟频率单位：MHz 频率(frequency)
    parameter MAX_TIME = 20; // 单位：ms，抖动判断时间，20ms？
    localparam TIMER_MAX_VAL = MAX_TIME * 1000 * FREQ; // ms(10^-3) * MHz(10^6) = 10^3 = 1000; 这里的值应该是最防抖过程中的最大计数次数

    reg [N-1:0] q_reg; //
    reg [N-1:0] q_next;
    reg DFF1, DFF2; // 输入 D 触发器
    wire q_add; // 标志位
    wire q_reset;
    reg button_out_d0;

    assign q_reset = (DFF1 ^ DFF2); // 异或运算，同则为0，不同则为1
    assign q_add = ~(q_reg == TIMER_MAX_VAL); // reg计数到 TIMER_MAX_VAL 则停止计数；


    always @ (q_reset, q_add, q_reg)
        begin
            case ({q_reset, q_add})
                2'b00:
                    q_next <= q_reg;
                2'b01:
                    q_next <= q_reg + 1;
                default :
                    // 复位
                    q_next <= {N{1'b0}};
            endcase
        end

    // 触发器输入，q值更新
    always @ (posedge clk or posedge rst)
        begin
            if (rst == 1'b1)
                begin
                    DFF1 <= 1'b0;
                    DFF2 <= 1'b0;
                    q_reg <= {N{1'b0}};
                end
            else
                begin
                    DFF1 <= button_in;
                    DFF2 <= DFF1;
                    q_reg <= q_next;
                end
        end

    // 计数控制 button_out
    always @ (posedge clk or posedge rst)
        begin
            if (rst == 1'b1)
                button_out <= 1'b1;
            else if (q_reg == TIMER_MAX_VAL)
                button_out <= DFF2;
            else
                button_out <= button_out;
        end

    // 计数控制 button_out_d0
    always @ (posedge clk or posedge rst)
        begin
            if (rst == 1'b1)
                begin
                    button_out_d0 <= 1'b1;
                    button_posedge <= 1'b0;
                    button_negedge <= 1'b0;
                end
            else
                begin
                    button_out_d0 <= button_out;
                    button_posedge <= ~button_out_d0 & button_out;
                    button_negedge <= button_out_d0 & ~button_out;
                end
        end

endmodule

最后是综合模块：

/**
* @Des:
* @Author: preta
* @Date: 2021-9-6
*/

module key_debounce (
    input sys_clk_p,
    input sys_clk_n,
    input rst_n,
    input key,
    output [3:0] led
);

    wire sys_clk;
    IBUFGDS u_ibufg_sys_clk (
        .I (sys_clk_p),
        .IB (sys_clk_n),
        .O (sys_clk)
    );

    wire button_negedge; // 键下降沿
    ax_debounce ax_debounce_m0 (
        .clk (sys_clk),
        .rst (~rst_n),
        .button_in (key),
        .button_posedge (),
        // 只获取按键的下降沿信号（按键按下），这里只对这个接口进行了赋值
        .button_negedge (button_negedge),
        .button_out ()
    );

    wire [3:0] count;
    wire t0;
    count_m10 count10_m0 (
        .clk (sys_clk),
        .rst_n (rst_n),
        .en (button_negedge),
        .clr (1'b0),
        .data (count),
        .t (t0)
    );
    assign led = count;

endmodule

在Alinx 7325开发板上将这个示例运行了。终于注意到实验现象是每按一次按键，实现计数器加一，并将结果以LED形式显示。
从0000加到1001（0~9）。这个开发板的例程好是好，但是就像这样，直到最后一步才会介绍实验的现象，给前期的代码理解
造成了一些不变。
不过也还好，因为最开始实验现象和例程中介绍的不一样，通过自己复写的代码和示例源码的比较，又发现了一些关于硬件理
解上的问题。在编写.xcd文件中，对两个按键进行引脚分配时，采用的 IOSTRANDARD 写错，导致代码烧录之后开发板的风扇
一直转动。还有，这块开发板的系统时钟是 200MHz，而可编程时钟源是 50MHz，实验中一般使用的 sys_clk_p 就是系统时钟源。
似乎涉及到时钟源的代码，每次都要使用到相关的 Vivado 原语。
